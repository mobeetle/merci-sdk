Lesson 6: The Parameter Builder Pattern

Learn how to control the model's output with advanced parameters like temperature and response format using the SDK's fluent builder pattern.

Code: lesson_6_parameter_control.mjs

The .withParameters() method accepts a function that receives a builder instance. You can chain methods on this builder to configure the request. The SDK intelligently filters any parameters not supported by the selected model.

import { MerciClient, createUserMessage } from '../lib/merci.2.11.0.mjs';
import { token } from '../secret/token.mjs';

async function runExperiment(client, prompt, builderFn, description) {
    console.log(`\n--- Experiment: ${description} ---`);

    // Configure the session with parameters using the builder pattern
    const chatSession = client.chat('google-chat-gemini-flash-2.5')
        .withParameters(builderFn);

    // ... stream processing logic ...
}

async function main() {
    const client = new MerciClient({ token });
    const storyPrompt = "Tell me a very short story about a robot who discovers music.";

    // Experiment 1: Focused and Deterministic (Low Temperature)
    await runExperiment(client, storyPrompt, builder => builder.temperature(0.1));

    // Experiment 2: Creative and Unpredictable (High Temperature)
    await runExperiment(client, storyPrompt, builder => builder.temperature(1.0));

    // Experiment 3: Forcing JSON Output
    await runExperiment(client, jsonPrompt, builder => builder.asJson());
}

main();

Key Concepts

Different parameters allow you to fine-tune the model's behavior:

  • .temperature(): Controls randomness. Lower values (e.g., 0.1) make the output more deterministic and focused, while higher values (e.g., 1.0) make it more creative and unpredictable.
  • .asJson(): Forces the model to output a valid, parsable JSON string, which is extremely useful for structured data extraction.
  • .topP() / .topK(): Other methods to control the sampling of the next word, offering alternatives to temperature for tuning creativity.
