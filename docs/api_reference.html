API Reference

This document provides a detailed reference for the Merci SDK's classes, methods, and types.

MerciClient

The main entry point for the SDK. It handles authentication, token refreshing, and serves as the factory for creating ChatSession instances.

Constructor

Initializes a new client instance.

new MerciClient(options: { token: string })

Parameters

  • options.token {string} - Required. Your JSON Web Token (JWT) for the JetBrains AI Platform.

Methods

.chat(profile)

Creates a new, configurable chat session for a specific model.

.chat(profile: string): ChatSession

Parameters

  • profile {string} - The identifier for the model to use (e.g., 'google-chat-gemini-flash-2.5', 'openai-gpt-5-mini').

Authentication & Token Management

The SDK handles token expiration and refreshing automatically. When an API request fails with a 401 Unauthorized status, the client transparently performs the following actions:

  1. Catches the 401 error.
  2. Emits the token_refresh_start event.
  3. Makes a request to the refresh endpoint to obtain a new JWT.
  4. If successful, it emits the token_refresh_success event with the new token.
  5. Automatically retries the original failed request with the new token.

This entire process is seamless, but you can use the events to monitor it and persist the new token.

ðŸ“š See Lesson 13: Automated Token Management for a practical example of this flow.

Events

The client is an EventEmitter that provides hooks for observability. Use client.on('eventName', callback) to listen.

  • api_request - Fired just before an API request is sent. Payload: { url, method, body }.
  • api_response - Fired when an API response is received. Payload: { url, status, ok }.
  • token_refresh_start - Fired when the SDK detects an expired token and begins the refresh process.
  • token_refresh_success - Fired after a new token has been successfully obtained. Payload: (newToken: string).
  • error - Fired when an API or SDK-internal error occurs. Payload: APIError instance.
  • tool_start - Fired when the .run() method begins executing tool calls. Payload: { calls: ToolCall[] }.
  • tool_finish - Fired when the .run() method finishes executing tool calls. Payload: { results: ToolExecutionResult[] }.
  • parameter_warning - Fired when a parameter is ignored because it's not supported by the selected model. Payload: { parameter, profile, message }.

Represents a configured chat session. This class uses a fluent, chainable interface for configuration before execution.

Execution Methods

.stream(input)

The primary, low-level method for executing a request. It returns an async iterator that yields events in real-time as they are received from the model. This is ideal for UIs that display a typing effect or for complex agentic loops that need to inspect the model's output piece by piece.

.stream(input: string | ChatMessage[]): AsyncIterable<StreamEvent>
ðŸ“š See Lesson 11: Multi-Turn Clarification for an advanced use case of manual stream handling.

.run(input)

A high-level "agent" method that automates the entire multi-step tool-use process. It handles the back-and-forth conversation with the model required to execute tools and synthesize a final answer. Use this for simplicity when you want a final text answer based on tool execution.

.run(input: string | ChatMessage[]): Promise<string>
ðŸ“š See Lesson 8: Basic Tool Usage for a clear example of the .run() method.

ParameterBuilder

A fluent class for setting advanced model parameters. An instance of this class is passed to the .withParameters() method.

.temperature(value: number): Controls randomness. Lower is more deterministic.
.topP(value: number): Nucleus sampling probability mass.
.topK(value: number): Considers the top K most likely tokens.
.length(value: number): Sets the maximum number of tokens to generate.
.stopToken(value: string): A sequence where the API will stop generating.
.seed(value: number): Makes the output deterministic for the same seed.
.asJson(): Forces the model to output a valid JSON object.
.parallelToolCalls(isEnabled: boolean): Allows the model to call multiple tools in parallel.
.toolChoiceAuto(): (Default) Model decides whether to call a tool.
.toolChoiceRequired(): Forces the model to call at least one tool.
.toolChoiceNone(): Prevents the model from calling any tools.
.toolChoiceNamed(name: string): Forces the model to call a specific tool.

Type Definitions

Key TypeScript interfaces that define the shape of data used throughout the SDK.

ToolDefinition

The structure for defining a tool the model can use.

interface ToolDefinition {
  name: string;
  description: string;
  parameters: object; // JSON Schema object
  execute: (args: any) => any | Promise<any>;
}

StreamEvent

A union of all possible event types yielded by the .stream() method.

type StreamEvent =
  | { type: 'text'; content: string }
  | { type: 'tool_calls'; calls: ToolCall[] }
  | { type: 'quota'; data: object };
