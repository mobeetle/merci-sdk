API Reference

This document provides a detailed reference for the Merci SDK's classes, methods, and types.

MerciClient

The main entry point for the SDK. It handles authentication, token refreshing, and serves as the factory for creating ChatSession instances.

Constructor

Initializes a new client instance.

new MerciClient(options: { token: string })

Parameters

  • options.token {string} - Required. Your JSON Web Token (JWT) for the JetBrains AI Platform.
ðŸ“š See Getting Started for setup instructions.

Methods

.chat(profile)

Creates a new, configurable chat session for a specific model.

.chat(profile: string): ChatSession

Parameters

  • profile {string} - The identifier for the model to use (e.g., 'google-chat-gemini-flash-2.5').
ðŸ“š See Lesson 1 (Basic Usage) and Lesson 3 (Model Selection).

Events

The client is an EventEmitter that provides hooks for observability. Use client.on('eventName', callback) to listen.

  • api_request / api_response - Monitor network activity.
  • token_refresh_start / token_refresh_success - Track automatic token refreshing.
  • error - Catch SDK-internal errors.
  • tool_start / tool_finish - Observe the lifecycle of agentic tool calls.
  • parameter_warning - Get notified when a parameter is ignored for a specific model.
ðŸ“š See Lesson 13 (Token Management) and Lesson 14 (Observability).

ChatSession

Represents a configured chat session. This class uses a fluent, chainable interface for configuration before execution.

Configuration Methods

.withTools(tools)

Equips the session with a set of tools the model can use. This is the foundation for building agents.

.withTools(tools: ToolDefinition[]): this
ðŸ“š See Lesson 8 (Basic), Lesson 9 (Parallel), Lesson 10 (Extraction), Lesson 17 (Multimodal), and Lesson 18 (Dynamic).

.withSystemMessage(content)

Configures a system message to guide the AI's behavior, setting its persona, rules, or context for the entire conversation.

.withSystemMessage(content: string): this
ðŸ“š See Lesson 2: System Messages for a practical example.

.withParameters(builderFn)

Configures advanced model parameters for the request using a fluent builder. This allows for fine-grained control over the model's output.

.withParameters(builderFn: (builder: ParameterBuilder) => ParameterBuilder): this
ðŸ“š See Lesson 6 (Parameter Control), Lesson 7 (JSON Output), and Lesson 16 (Forcing Tool Usage).

Execution Methods

.stream(input)

The primary, low-level method for executing a request. It returns an async iterator that yields events in real-time. This is ideal for UIs that display a typing effect or for complex agentic loops that need to inspect the model's output piece by piece.

.stream(input: string | ChatMessage[]): AsyncIterable<StreamEvent>
ðŸ“š See Lesson 11 (Clarification), Lesson 12 (Parallel Extraction), and Lesson 15 (Manual Loop).

.run(input)

A high-level "agent" method that automates the entire multi-step tool-use process. Use this for simplicity when you want a final text answer based on tool execution.

.run(input: string | ChatMessage[]): Promise<string>
ðŸ“š See Lesson 8: Basic Tool Usage for a clear example.

Type Definitions

Key TypeScript interfaces that define the shape of data used throughout the SDK.

ToolDefinition

The structure for defining a tool the model can use.

export interface ToolDefinition {
  name: string;
  description: string;
  parameters: object; // JSON Schema object
  execute: (args: any) => any | Promise<any>;
}

StreamEvent

A union of all possible event types yielded by the .stream() method.

export type StreamEvent =
  | { type: 'text'; content: string }
  | { type: 'tool_calls'; calls: ToolCall[] }
  | { type: 'quota'; data: object };
