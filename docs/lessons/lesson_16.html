Lesson 16: Forcing Tool Usage

Learn how to make agent behavior more reliable and predictable by forcing the model to use a provided tool, rather than leaving the decision up to its discretion.

Code: lesson_16_forcing_tool_choice.mjs

This script runs two experiments. The first is a standard agent where tool use is optional. The second uses the parameter builder to set .toolChoiceRequired(true), which constrains the model to call one of the available tools.

// lesson_16_forcing_tool_choice.mjs
import { MerciClient } from '../lib/merci.2.11.0.mjs';
import { token } from '../secret/token.mjs';

const databaseTool = { /* ... */ };

async function main() {
    const client = new MerciClient({ token });
    const tools = [databaseTool];

    // --- EXPERIMENT 1: Optional Tool Usage ---
    console.log('\n--- EXPERIMENT 1: Optional Tool Usage ---');
    const optionalAgent = client.chat(MODEL).withTools(tools);
    const optionalResponse = await optionalAgent.run("Can you check the database for me?");
    console.log(`ðŸ¤– Assistant > ${optionalResponse}`);

    // --- EXPERIMENT 2: Required Tool Usage ---
    console.log('\n--- EXPERIMENT 2: Required Tool Usage ---');
    const requiredAgent = client
        .chat(MODEL)
        .withTools(tools)
        .withParameters(builder => builder.toolChoiceRequired(true)); // The key parameter

    const requiredResponse = await requiredAgent.run("Find all active users in the database.");
    console.log(`ðŸ¤– Assistant > ${requiredResponse}`);
}

main();

Expected Output

In the first experiment, the model might simply respond conversationally without using the tool. In the second, it is forced to call the query_user_database tool, leading to a data-driven response.
