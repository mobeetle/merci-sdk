Lesson 15: The Manual Tool-Use Loop

This lesson deconstructs the high-level .run() agent to reveal the fundamental, multi-step process of tool use, giving you maximum control over the interaction.

Code: lesson_15_manual_tool_loop.mjs

Instead of .run(), this script uses .stream() twice. The first call gets the model's request to use a tool. We then execute the tool, add the results to the message history, and make a second .stream() call to get the final, natural language summary.

// lesson_15_manual_tool_loop.mjs
import { MerciClient, executeTools, /* ...message helpers... */ } from '../lib/merci.2.11.0.mjs';

async function main() {
    const chatSession = client.chat(MODEL).withTools([weatherTool]);
    const messages = [createUserMessage("What's the weather in Paris?")];

    // 1. First API call to get the tool request
    let toolCalls = [];
    for await (const event of chatSession.stream(messages)) {
        if (event.type === 'tool_calls') toolCalls = event.calls;
    }

    // 2. Execute the tool locally
    const toolResults = await executeTools(toolCalls, [weatherTool]);

    // 3. Update history with tool call and result messages
    messages.push(createAssistantToolCallMessage(...));
    messages.push(createToolResultMessage(...));

    // 4. Second API call to get the final summary
    let finalResponse = '';
    for await (const event of chatSession.stream(messages)) {
        if (event.type === 'text') finalResponse += event.content;
    }
    console.log(finalResponse);
}

main();

Expected Output

The manual loop makes the agent's process explicit. The output shows the initial user prompt, the log from the local tool execution, and finally the model's summarized answer based on the tool's result.
